Pengenalan: Apa itu Wagtail?

Wagtail adalah Content Management System (CMS) sumber terbuka (open-source) yang ditulis dengan Python dan dibangun di atas framework Django. Filosofi utamanya adalah memberikan kebebasan penuh kepada developer sambil menyediakan antarmuka yang intuitif dan menyenangkan bagi penulis konten. Wagtail digunakan oleh organisasi besar seperti NASA, Google, dan NHS (Layanan Kesehatan Nasional Inggris).

Fitur Unggulan

  • Arsitektur Fleksibel: Dibangun di atas model Django.
  • StreamField: Pembuatan konten halaman yang dinamis dan modular.
  • Antarmuka Admin Modern: Bersih, cepat, dan ramah pengguna.
  • Manajemen Media Kuat: Termasuk cropping dan focal point.
  • Kinerja & Skalabilitas: Mewarisi kekuatan dari Django.
  • Komunitas Aktif: Ekosistem paket pihak ketiga yang kaya.

Instalasi dan Pengaturan Proyek

Memulai proyek Wagtail sangatlah mudah. Bagian ini akan memandu Anda melalui langkah-langkah dasar untuk menyiapkan lingkungan pengembangan lokal Anda, mulai dari instalasi hingga menjalankan server.

Langkah-langkah Memulai

  1. 1. Buat dan Aktifkan Virtual Environment:
    Bash 
    python -m venv wagtail_env
source wagtail_env/bin/activate
  2. 2. Instal Wagtail & Buat Proyek Baru:
    Bash 
    pip install wagtail
wagtail start myproject
  3. 3. Migrasi Database & Buat Superuser:
    Bash 
    cd myproject
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser
  4. 4. Jalankan Server Pengembangan:
    Bash 
    python manage.py runserver

    Antarmuka admin akan tersedia di http://127.0.0.1:8000/admin

Struktur Proyek yang Dihasilkan

myproject/

├── home/ # Aplikasi untuk halaman beranda

├── search/ # Aplikasi untuk pencarian

├── myproject/ # Pengaturan proyek

└── manage.py # Skrip utilitas Django

Konsep Inti: Page Models

Semua konten di Wagtail terstruktur dalam sebuah **pohon (tree)**. Setiap simpul adalah sebuah **Page**. Setiap tipe halaman (misalnya, Halaman Beranda, Halaman Blog) didefinisikan sebagai *model* Python yang mewarisi `wagtail.models.Page`. Anda dapat mengontrol hierarki dengan `parent_page_types` dan `subpage_types`.

Contoh Model Halaman Blog

blog/models.py 
from django.db import models
from wagtail.models import Page
from wagtail.fields import RichTextField
from wagtail.admin.panels import FieldPanel, MultiFieldPanel

class BlogPage(Page):
    author = models.CharField(max_length=255)
    date = models.DateField("Post date")
    body = RichTextField(blank=True)

    # Hanya bisa dibuat di bawah Halaman Daftar Blog
    parent_page_types = ['blog.BlogIndexPage'] 
    # Tidak bisa memiliki anak
    subpage_types = []

    content_panels = Page.content_panels + [
        MultiFieldPanel([
            FieldPanel('author'),
            FieldPanel('date'),
        ], heading="Informasi Postingan"),
        FieldPanel('body'),
    ]

Konsep Inti: StreamField

StreamField adalah fitur paling kuat di Wagtail untuk konten yang fleksibel. Ini membebaskan Anda dari batasan satu field `RichTextField` monolitik, memungkinkan editor untuk membangun halaman dari berbagai jenis "blok" konten dalam urutan apa pun.

Contoh Implementasi

models.py 
from wagtail.fields import StreamField
from wagtail.blocks import CharBlock, RichTextBlock, StructBlock
from wagtail.images.blocks import ImageChooserBlock

class ArticlePage(Page):
    body = StreamField([
        ('heading', CharBlock(icon="title")),
        ('paragraph', RichTextBlock(icon="pilcrow")),
        ('image', ImageChooserBlock(icon="image")),
        ('quote', StructBlock([
            ('text', TextBlock()),
            ('author', CharBlock()),
        ], icon='openquote'))
    ], use_json_field=True, blank=True)

Konsep Inti: Gambar, Snippets & Forms

Gambar dan Dokumen

Wagtail memiliki aplikasi bawaan untuk mengelola aset media. Saat menggunakan gambar di template, Anda dapat memanipulasinya dengan tag `{% image %}`.

Django Template 
{% load wagtailimages_tags %}

{# Mengubah ukuran lebar gambar #}
{% image page.header_image width-400 %}

{# Mengisi dan memotong gambar #}
{% image page.header_image fill-300x200 %}

Snippets

Snippets adalah potongan konten yang dapat digunakan kembali dan tidak hidup di dalam pohon halaman. Ideal untuk data seperti Kategori, Tag, atau Profil Penulis.

Forms

Wagtail memudahkan pembuatan formulir kontak atau formulir lainnya dengan mewarisi `AbstractEmailForm`.

Templating

Wagtail menggunakan sistem template Django. Selain tag dan filter standar Django, Wagtail menyediakan beberapa tag yang sangat berguna untuk merender kontennya.

Tag Template Wagtail yang Umum

{% pageurl page_object %}

Menghasilkan URL untuk sebuah objek halaman.

{% slugurl "page_slug" %}

Menghasilkan URL halaman berdasarkan slug-nya.

{% image image_object [filter] %}

Merender dan memanipulasi gambar.

|richtext

Merender konten dari `RichTextField` dengan aman.

{% include_block block_value %}

Cara yang lebih bersih untuk merender blok StreamField menggunakan template terpisah.

Topik Lanjutan

Setelah menguasai dasar-dasarnya, Anda dapat menjelajahi fitur-fitur canggih Wagtail untuk membangun aplikasi yang lebih kompleks dan disesuaikan.

Kustomisasi Antarmuka Admin

Gunakan **Hooks** untuk menyisipkan CSS/JS kustom, dan **ModelAdmin** untuk antarmuka CRUD pada model non-halaman.

Pencarian (Search)

Integrasi dengan **Elasticsearch** untuk pencarian yang cepat dan kuat. Konfigurasikan `search_fields` pada model Anda.

API Headless

Gunakan API v2 bawaan untuk mengekspos konten sebagai JSON untuk aplikasi front-end (React, Vue, dll.).

Multi-Situs dan Internasionalisasi (i18n)

Jalankan beberapa situs dari satu instalasi dan sediakan konten dalam berbagai bahasa.

Manajemen Pengguna & Izin

Buat **Grup** pengguna dan berikan izin spesifik pada bagian-bagian tertentu dari pohon halaman.

Deployment

Praktik deployment Wagtail sama dengan aplikasi Django pada umumnya. Berikut adalah daftar periksa penting untuk memastikan proses deployment ke lingkungan produksi berjalan lancar dan aman.

Daftar Periksa Produksi

  • Atur DEBUG = False untuk keamanan.
  • Muat SECRET_KEY dari *environment variable*.
  • Konfigurasikan ALLOWED_HOSTS dengan domain Anda.
  • Gunakan **PostgreSQL** sebagai database di produksi.
  • Jalankan python manage.py collectstatic.
  • Gunakan layanan seperti **Amazon S3** untuk file media.
  • Gunakan **Gunicorn** atau uWSGI sebagai server aplikasi.
  • Gunakan **Nginx** sebagai *reverse proxy*.
